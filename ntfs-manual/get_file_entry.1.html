NAME

get_file_entry - read NTFS file record at MFT address

SYNOPSIS

#include <file_entry.h> /* See NOTES */

file_entry get_file_entry(unsigned long long mft_address);

void free_file_entry(file_entry file);

Compile and link with -DSATA_MEDIATOR when cross-compiling for SATA Mediator hardware.

DESCRIPTION

get_file_entry() reads the sectors corresponding to the NTFS file record at the mft_address and returns the file_entry union defined in file_entry.h :

typedef union {
  file_header * header;
  unsigned char * content;
} file_entry;

The file_header struct represents the NTFS file record header and is also defined in file_entry.h.

typedef struct file_header {
  unsigned char magic_signature[4];
  unsigned short update_seq_offset;
  unsigned short size_seq_number_array; 
  unsigned long long LSN;
  unsigned short seq_number;
  unsigned short hard_link_count;
  unsigned short first_attr_offset;
  unsigned short flags;
  unsigned int used_file_record_size;
  unsigned int allocated_file_record_size;
  unsigned long long file_ref_to_base_file_rec;
  unsigned short next_attr_id;
  unsigned short align;
} file_header;

Each call to get_file_entry() allocates memory for the file_header * and unsigned char * by calling malloc from stdlib.h. It initialize the unsigned char * content member of the union to the size of the NTFS file record which is obtained using the FILE_RECORD_SIZE_IN_BYTES macro defined in volume_info.h.

This memory can then be freed by calling free_file_entry().

get_file_entry() is dependendent upon a preceding call to initialize_file_system() found in initialize.h

RETURN VALUE

On success get_file_entry returns a file_entry union containing the file_header * header corresponding to the NTFS file record header info and unsigned char * content containing the entire NTFS file record. On failure the union is returned with both pointers set to NULL.

NOTES

All headers for the SATA NTFS project can be found in the include directory in the project repo. Compile with -I./include/.
